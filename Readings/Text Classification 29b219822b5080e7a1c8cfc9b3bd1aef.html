Classification task

text classification은 결국에 data를 분석하고 label을 붙이는데에 중요함 ⇒ 앞에서 text를 vector로 만들었고 해당 vector들을 이용해 classification을 수행 ⇒ pretrain & fine-tune 을 이용해 일반적인 언어의 지식(pretrain)을 특정한 classification task(fine tune)에 사용할 수 있도록 만든다

Classification

pretraining

  • motivation: 실세계에는 많은 양의 text data와 언어학적 패턴이 존재
  • goal: human labelling없이 일반적인 representation을 학습
    • Word2Vec / Bert

fine-tuning

  • motivation: pretrained model은 일반적인 지식을 포착하지만, classification과 같은 작업은 구체적인 knowledge가 필요로 함
  • goal: pretrained model의 parameter을 특정 downstream task에 알맞도록 학습
    • Sentiment classification using Bert

pretraining & fine-tuning

  • pretraining: 좋은 initialization ⇒ SGD 는 initialization에 큰 영향을 받음 ,
  • fine tuning: pretrained model을 특정 task에 사용이 가능하도록 학습

👍 효율적인 수렴 & 더 적은 data를 가지고 일반화 가능

ex) pretrained BERT ⇒ 1) CLS Representation 2) Average all contextual embedding을 represent vector로 선택 ⇒ small classification head 추가 ⇒ labeled data로 fine-tuning

  1. full fine-tuning : 모든 parameter 업데이트
  1. partial fine-tuning: 일부분의 parameter만 업데이트 (head만 update 시키거나 encoder에 일부분만 update)

classification task

  1. Binary classification ⇒ Bernoulli RV → Binary Cross Entropy loss function
  1. Multi-class classification ⇒ Categorical RV → Cross Entropy loss function
  1. Multi-label classification ⇒ independent Bernoulli RV → BCE loss function independently per class
    • 각 logit마다 각각 sigmoid를 사용 ⇒ softmax를 사용하지 않음 (다른 class에 영향을 미치지 않음)
    • 각 class마다 threshold를 적용하여 해당 class에 적용되는지 확인

Evaluation

  • accuracy
    • 👎 class가 imbalance하거나 희귀할 때 문제 ⇒ accuracy는 높아보일 수 있어도 제대로 detect하지 못함
  • precision : TP/(TP+FP) ⇒ Positive라고 예측한 것 중에 실제 Positive일 확률
  • recaㅣl : TP/(TP+FN) ⇒ 실제 Positivie 중에 Positive라고 예측한 확률
    • 일반적으로 precision과 recall은 trade-off 관계를 가짐 → threshold의 sensitivity를 조절해보면 알 수 있음!
    • threshold가 높을 때 ⇒ precision이 좋음 (내가 예측한 것 대부분이 실제 positive)
    • threshold가 낮을 떄 ⇒ recall이 좋음 (실제 positive가 내 예측에 들어감)
  • f1 score: 2PR/(P+R) ⇒ Precision과 Recall 모두 다 높을 때만 높음
    • 비율간의 평균으로 생각! ⇒ 속도- 시간 예시를 생각해보면 좋음 (둘다 분자 TP는 같고 분모가 다른 경우)

multi-class classification

class가 C개일때는 confusion matrix가 CXC table이 되며 이때 diagonal cell 이 각 class마다 TP가 된다. ⇒ 해당 confusion matrix를 특정 class에 대한 confusion matrix로 설계가 가능

🤔 각 class마다 accuracy, recall, precision이 나오는데 우리가 원하는 건 전체적인 성능을 보여주는 단일 metric!

  1. Macro averaging : 각 class마다 계산 후 평균 ⇒ class size와 상관없이 모든 class를 동일하게 고려
    • 👍 모든 class를 동일하게 고려 ⇒ 소수 class 대한 성능도 강조
    • 👎 소수 class에 대해서 성능 안좋음 ⇒ 전체적인 성능이 과소 평가
  1. Micro averaging: 모든 class에서 TP, FP, FN을 구한 뒤에 precision, recall, f1을 계산 ⇒ 빈도수에 따라 class에 weight를 부여
    • 👍 imbalance에 강건하며 전체적인 성능을 반영
    • 👎 다수 class에 의해 값이 결정되어, 소수 class에 대한 성능 반영이 안됨

⭐️ class imbalance가 걱정이 될 경우 ⇒ Macro , 전체적인 성능이 중요 ⇒ Micro

지금까지는 모든 data에 label이 있는 경우에 대해서만 다룸 ⇒ 그렇다면 모든 data instance에 label이 존재할까?

Learning with limited labels

실제 세계에서는 data instance들에 대해 label이 없는 경우가 매우 많다 ⇒ 이럴 때 어떤 방법을 사용할 수 있을지 알아볼 예정!

  1. Semi Supervised learning : 어떻게 unlabeled data들을 효과적으로 이용할 수 있을까?
  1. Multi-task learning: 특정 task에 label이 부족할 경우, 유사한 task에서 signal을 빌릴 수 있지 않을까?
  1. Adversarial learning : 만약에 labeled data와 unlabeled data가 다른 분포에서 온 것이면 어떻게 할까?

1. Semi-Supervised learning

Semi-supervised learning : labeled와 unlabeled 된 data 모두를 이용하여 학습을 진행

⇒ 적은 수의 data들로 supervsied learning을 했을 때 일반화가 되지 않을 수 있음 (최적의 decision boundary 찾기 힘듦)

  • labeled data : supervision에 대한 정보를 제공
  • unlabeled data : 모델이 일반화가 되도록 도와줌

⭐️ 주요 assumption!

  1. smoothness assumption : input space에서 가까이 있는 data point는 같은 label일 것 이다!
  1. Low-density assumption: 좋은 decision boundary는 많은 데이터들이 포함되어 있는 지역을 피해야한다! (위 그림 처럼 data point들이 모여있지 않은 low-density area를 지나야한다)

1-1. Pseudo labeling

Pseudo labeling : confident prediction을 이용하여 unlabeled data에 label을 할당하는 방식

  • label data로 학습이 되는 동안에 model은 점점 특정 unlabeled sample에 대해서 confident해지게 됨

process

  1. label data로 학습 (맨 처음에는 일부분의 label data만 존재 ⇒ supervised way로 학습)
  1. unlabel data에 대해서 label 예측 ⇒ 여러가지 confidence level로 예측을 진행
  1. pseudo-labeled data 생성 ⇒ 특정 confidence를 넘는 data에 대해서 실제 true-labeled sample로 다룸
  1. model retrain

👍 간단하고 효과적인 방식

👎 문제점

  • 사람이 label을 만든 것보다 정확하지 않음 ⇒ 오히려 labeled data만으로 학습한 것보다 성능이 안좋을 수 있음 & local pattern 만을 포착
  • error propagation: 잘못된 pseudo-label이 계속해서 존재할 수 있음
  • overconfidence: 잘못된 예측임에도 모델이 매우 confident 할 수 있음

1-2. Consistency regularization

consistency regularization : 같은 input에 조금만한 변화가 있더라도 모델은 계속해서 일관된 예측을 해야함 ⇒ small perturbation이 있더라도 예측은 안정적이어야함

  • embedding에 random noise 추가
  • word dropout or masking
  • data augmentation

⭐️ regularization 항을 추가하여 모델이 perturbation이 있더라도 일관된 예측을 하도록 학습을 진행

  1. regularization 효과로 noise에 강건해져 overfitting을 막을 수 있다 (perturbation이 있을 때 결과가 크게 달라지면 해당 모델이 불안정 ⇒ overfitting)
  1. pseudo labeling 이 오직 high-confidence sample만을 사용하는 반면에 모든 unlabeled data를 사용 ⇒ dataset에 더 많은 부분을 사용하게 됨
    • unlabel sample이라고 하더라도, 해당 sample에 perturbation을 준 것과 비슷한 확률 분포를 가지게 만들도록 하기 때문에 모든 unlabeled data들을 사용한다고 볼 수 있음

👎 다만, 모델이 unlabeled data들에 대해서 불안정한 예측을 내릴 수 있음 ⇒ 해당 data들이 gold label이 없으므로 epoch이 증가하면서 확률 분포가 계속해서 바뀔 수 있다

1-3. Consistency regularization- Temporal ensemble

🤔 그렇다면 불안정한 unlabeled data들에 대해 어떻게 model 예측을 안정화시킬 수 있을까?

🅰️ model ensemble : 하나의 단일 모델이 아닌 여러 모델을 학습시킨 후 예측을 합치는 방법 ⇒ variance가 감소하고 더 안정적인 예측이 가능해짐

👎 하지만 여러 모델들을 학습시키는 건 비쌈

💡 하나의 모델에서 서로 다른 epoch에서의 결과를 모아서 ensemble을 만들자! ⇒ model ensemble과 같은 방식!

⇒ model parameter들을 EMA(exponential moving average) 를 통해 ensemble을 진행

즉, 1) 현재 model과 2) temporal ensemble model 2개가 존재하게 되며, temporal ensemble model은 현재 모델을 느리게 따라가면서 안정적으로 학습하게 된다

👍 이렇게 함으로써 unlabled data에 대해서도 안정적으로 학습이 가능하게 만들 수 있다

  • effective use of unlabeled data: perturbation만 추가하여 unlabled data에 대해서 폭넓게 적용 가능
  • simple concept : 해당 모델에 regularizer만 추가

👎 그럼에도 본질적인 문제점들이 존재

  • perturbation sensitivity: noise나 augmentation을 어떻게 하는지에 따라 성능이 달라지게 됨
    • CV에서 crop은 효과적일 수 있지만 text에서는 효과적이지 않을 수 있음 (like→dislike가 되면 문맥이 완전히 달라짐)
  • weak direct guidance : label이 제공되지 않기 때문에 여전히 boundary가 불확실함

2. Multi-task learning (MTL)

이번에는 단순히 unlabeled data를 어떻게 효과적으로 사용할 것인지 알아보는 것이 아닌, 하나의 task에는 충분한 label이 없을 때 다른 task에서의 signal을 가져와서 학습을 진행할 수 있는 방법에 대해서 알아보고자 함!

ex) 추천시스템에서 2가지 예측 task를 진행하고자 함 ⇒ engagement , satisfaction (사용자가 item을 click하는지 / 사용자가 해당 item에 대해 높은 평점을 주는지)

  • 두가지 모델을 각각 학습시킬 수 있지만…
  • Assumption: item을 클릭한다는 것 자체가 이미 관심이 있기 때문에 만족할 가능성이 높다고 볼 수 있다! ⇒ 즉, 하나의 task를 해결하는 것이 다른 task를 해결하는데 도움이 될 수 있다!!

결국 두 task가 independet 하지 않을 때, 하나의 단일 모델을 사용하여 label이 희소하게 있는 문제를 해결할 수 있음 (shared parameter)

  • 👍 다양한 task의 supervision(label data)들을 사용 가능
  • 👍 generalization 향상과 label sparsity를 줄이는데 도움이 됨

👎 negative transfer : 지식을 서로 공유한느 것이 오히려 성능이 안좋아지는 현상을 의미하며, 때로는 각각 독립적으로 모델을 만드는 것이 더 효과적일 수 있음

  1. Task discrepancy : task들이 관련이 있다고 해도 여전히 차이점이 존재 ⇒ task들이 여러개 있을 때 특정 task들끼리 무관하거나 혹은 아예 상출될 수 있음
    • 💡 task similarity를 통해 parameter sharing의 정도를 조절 ⇒ MMoE
  1. optimization challenge: 특정 task는 빠르게 학습이 되어 shared parameter의 대부분에 영향을 미칠 수 있음 ⇒ 빠르게 학습이 되는 task에 대해 편향이 생겨, 느린 task에 대한 학습을 방해
    • 💡 balancing coefficient를 추가해 task의 학습 속도를 조절 ⇒ GradNorm

2-1. Multigate Mixture of Experts (MMoE)

https://dl.acm.org/doi/pdf/10.1145/3219819.3220007 : Multitask learning에서 task discrepancy에 따른 negative transfer 문제를 해결하기 위해 parameter sharing 정도를 조절하는 방법론

🤔 그렇다면 어떻게 공유하는 정도를 자동적으로 조절할 수 있을까?

하나의 큰 large network 대신에 여러개의 작은 subnetwork를 만들며 이를 expert 라고 칭함 ⇒ 각각은 input의 다른 측면에 집중을 함

  • 해당 expert 들은 weighted sum으로 결합되며, 해당 weight들은 각각의 expert들이 해당 task를 다루는데에 있어 얼마나 중요한지를 보여줌
  • 해당 weight 는 gate network 에 의해 결정이 되며, 각각의 gate는 expert들의 weight들을 softmax를 취해 확률적으로 만드는 작은 network를 의미한다

이후 학습은 기존 방식과 동일하게 진행된다 ⇒ 각 task마다 cross entropy loss를 취해 최적화 진행

🤔 1. 그렇다면 해당 MMoE 구조가 어떻게 task discrepancy를 줄일 수 있을까?

🅰️ 각 expert 는 입력의 서로 다른 측면에 집중을 하며, 각각의 task specific gate 는 각각의 task에 대해 얼마나 유용한지를 기반으로 expert output을 결합

  • 만약 두 task가 서로 비슷하다면, gate에서의 softmax distribution이 서로 비슷 ⇒ sharing more
  • 만약 두 task가 서로 덜 비슷하다면, gate에서의 softmax distribution이 서로 다르게 됨 ⇒ sharing less

🤔 2. 왜 각각의 expert 들은 학습이 진행됨에 따라 전문화가 되는 것일까?

🅰️ 각각의 task specific gate 는 task loss를 가장 효과적으로 줄일 수 있게 만드는 expert에 더 많은 weight를 주게 됨 ⇒ 특정 expert들이 특정 task에 대해 매우 많이 이용이 되게 됨

2-2. GradNorm (Adaptive loss balancing for MTL)

https://arxiv.org/abs/1711.02257 : 각 task마다 학습 난이도가 달라 학습 속도에 차이가 존재 ⇒ 다른 task 학습 속도 저하 및 parameter 이용에 문제를 만들 수 있음

💡 각 task마다 balancing coefficient 를 이용해 조절을 진행 ⇒ 학습속도가 빠르면 coefficient를 줄임 (weight 줄임)

결국 중요한건 전체 task 중에서 해당 task가 얼마나 빠르고 느린지이기 때문에 분모에 평균값이 들어가게 됨

⇒ 가장 쉽게 task 속도를 조절하는 방식: λi=λi∗rI\lambda_i = \lambda_i*r_Iλi​=λi​∗rI​﻿ (다른 방식도 있지만 해당 수업에서는 다루지 않음)

결국 Ltotal=λ1L1+λ2L2L_{total}=\lambda_1L_1+\lambda_2L_2Ltotal​=λ1​L1​+λ2​L2​﻿ (각 task에 대한 loss가 존재함) 이 됨

만약 L1의 gradient norm이 훨씬 클 경우 Task A가 shared parameter을 거의 독점 ⇒ L2의 gradient가 작아서 shared parameter에 거의 영향이 없게 됨 : negative transfer!

각 task의 shared parameter에 대한 gradient norm 을 구한 뒤에 lambda의 값을 조절해 G1=G2가 되도록 만들어 학습 속도를 일정하게 만드는 역할을 수행!

⭐️ 즉, 위에 사진에서 loss가 얼마나 줄었는지를 가지고 lambda의 값을 만드는데 이를 gradient norm 으로 측정해서 학습을 시킨다고 보면 됨!

3. Domain Adaptation

motivation example: Fake News detection! ⇒ news data와 label을 통해 모델 학습

⇒ 새로운 event들이 계속해서 들어오게 되는데, 이는 모델이 과거 data에 의해 학습이 되었기 때문에 새로운 event에 대해 일반화하는데 실패할 수 있음

⇒ 또한 이러한 새로운 news data들에 대해 label을 계속해서 만드는 것이 불가능함

problem adapttion : 위의 문제를 의미 ⇒ 우리는 다른 분포(domain)에서 온 두 data 집합을 가짐

  • labeled source domain : 해당 labeled source data에 대해서는 error를 최소화가 가능
  • unlabeled target domain : labeled source로 학습한 모델은 target domain 에 적용이 불가능할 수 있음 (데이터 분포 자체가 다를 수 있음)
    • SSL 방법 (pseudo labeling)을 적용할 수 있지만, 이러한 예측은 부정확하거나 편향될 수 있음

Assumption: 두 domain(distribution) 사이에는 x에서 y로의 mapping이 되는 공통된 decision pattern이 존재한다고 가정! ⇒ domain이 다르더라도 labeling rule 은 같다!

  • 항상 맞는 것은 아니지만 실제 많은 경우 해당 가정이 잘 들어맞는다

3-1. Data reweighting

source sample 각각이 target domain 에 얼마나 중요한지 측정을 한 뒤에 이를 각각 반영한다 ⇒ 즉, target distribution과 유사한 source example에 대해 가중치를 준다

Assumption: 두 domain 사이에 동일한 decision pattern이 존재할 것이다

⭐️ target domain에 대한 error를 최소화하는 식을 전개한 뒤, bayes rule을 사용하게 되면, 결국 target domain에 대해 error를 최소화하는 것은 binary classifier 로 계산이 가능해진다!

  • sample이 target domain일 확률이 높을 수록 w는 증가
  • sample이 source domain일 확률이 높을 수록 w는 감소

⭐️ 따라서 source sample과 target sample을 구별하는 binary classifier을 학습시켜 source data 에 대한 importance weight를 계산한 뒤, weighted loss function을 이용해 target classifier 을 학습시킨다!

⇒ Target domain과 분포가 비슷한 source sample에 더 높은 loss weight를 줘서, 모델이 target domain에 더 잘 일반화되도록 만들게 됨으로써 target sample이 오더라도 제대로 labeling을 진행

  • 여전히 source data만 학습에 사용

3-2. Adversarial Learning

https://arxiv.org/pdf/1505.07818

결국 위에서 알 수 있는 것은, ⭐️ 두 domain의 feature space가 제대로 align 되어있다면 일반화를 향상시킬 수 있다!

💡 그렇다면 두 domain의 feature distribution을 명시적으로 align시키도록 학습하면 되지 않을까?

현재 상황은 다음과 같다. feature extractor 에서 나온 결과로 task 1은 Fake/ Real (labeling)을 task 2 는 Source/ Target (weight)를 학습하는데에 입력으로 사용 ⇒ 해당 feature은 task 1,2에 대한 정보 둘 다 가지고 있다!

근데 task 2에 경우는 source와 target을 구별하는 것이지만 우리가 원하는건 source와 target의 구분을 feature space 상에서 없애는 것이기 때문에 Gradient Reversal Layer 이 필요!

GRL과 함께 learning objective과 다음과 같이 변화하게 됨

  • label classification loss : model이 main task를 올바르게 예측하도록 만듦
  • domain classification loss : source와 target domain을 구별하고자 노력
    • 앞에 -가 붙여있기 때문에 feature extractor는 domain confusion을 하도록 만들어짐

⭐️task 2 head 는 domain classification을 제대로 진행하게 만듦 ⇒ 이 head가 제대로 학습이 되어야만 올바른 reverse gradient가 feature extractor에 전달되게 됨

  • 제대로 학습이 되지 않을 경우 training collapse가 가능 ⇒ feature extractor에 의미있는 gradient를 더 이상 제공하지 않음
  • parameter가 다 0이 될 경우 classification을 못하므로 처음부터 classification loss를 최대화하고자 하면 안됨

Adversarial learning을 통해 feature alignment가 진행되게 되며 이를 통해 domain-invariant representation을 만들 수 있다!

👍 간단하고 효과적이며 source-target overlap에 less dependent (align을 하다록 학습이 진행됨

👎 global alignment : domain을 globally하게 정렬시키기 때문에 class 구조를 무시 ⇒ class misalignment가 생길 수 있음 ⇒ 서로 다른 class가 제대로 구별이 되어있다가 adversarial learning 이후에 class 구조가 무너질 수 있음
